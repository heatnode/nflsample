Dev Notes

I decided to use an angular/bootstrap/nvd3 stack for the application. Angular and bootstrap have proven to be a fairly rapid way to get a decent small scale application or prototype going quickly. Even though this is a prototype, I tried to keep the code reasonably well-organized and normalized. Here are some notes that may help:

Most of the relevant code is in a few places:

  • main.js - sets up the angular app
  • index.html - starting point and nav
  • /services/*.js - this is where most of the service stuff happens, charts, "data shaping", stat calculation, data retrieval
  • /controllers/QBPerformanceCtrl.js - has all the "view logic". In a real application, I would break this down further into either some additional directives or controllers
  • /views/qbperformance.html - has the primary view. Again, I would break this down into smaller pieces if this was a long term application

Other general notes

  • I have only done a tiny amount of d3 in the past so much of my effort was spent trying to work with it. I chose nvD3 and an angular directive based on nvD3 as a way to get up and running quickly.

  • There are not of ton of comments. I hope the code is somewhat readable and self-documenting. I try to write long and meaningful function and variable names.
